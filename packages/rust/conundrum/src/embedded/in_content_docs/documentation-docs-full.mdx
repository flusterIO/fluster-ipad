<AutoInsertedHeading depth={3} id="fluster" subtitle={null} >Fluster</AutoInsertedHeading>

Fluster is a note taking application that was built to suite my own personal note taking requirements while I was pursuing a <AutoInsertedMarkdownLink data={{"text":"modified model of relativity","url":"https://flusterapp.com/docs/user"}} />. While it was originally built as a `.mdx` application, it's syntax has grown to support even more than that.

With the ability to keep your hand-drawn notes linked to your AI-friendly markdown notes in a format that can expressive, yet transpile to the simple formats that AI and other platforms deal with seamlessly, I don't think there are any other applications like Fluster available. I built Fluster in the first place because none of the existing applications met my requirements, and I imagine that many users feel the same. While no application will ever be perfect, Fluster takes a simple approach: Separate your inputs, but keep them linked. Draw things that need to be drawn, type things as much as you can if you incoprorate a lot of AI into your workflow, and Fluster will help you find <span className="italic">both</span> types of content. These are some of the features that inspired the creation of Fluster:

- Search by:
  - Tag
  - Topic
  - Subject
  - Citation
  - Equation (Feature currently missing after migration, but will be back!)
- Embed:
  - Images (Currently remote only supported. As soon as I can afford internet and a real device I can work on the embedded media.)
  - Video (See above)
  - Gltf (3d-images, also... see above)
- Be more expressive, find what you're looking for with:
  - Cards, subtle, tinted backgrounds behind your content.
  - 'Admonitions', a more styled and flashy Card that can be used to draw attention to content.
  - 'Grid' to create responsive layouts easily that look good on any screen size.
  - 'Containers' as a blank-canvas, with dozens of accepted properties to create the layout you need.
  - Plus all of the class markdown features <span className="font-bold">with</span> a special markdown-extension syntax for things like a horizontal-rule that accepts child-text (look below) or a tag that is automatically associated with your note.



<Hr>App Usage Suggestions</Hr>

- The first should go without saying, but remember the documentation commands! Just `Docs??` to get you back to the root, on a new line all by itself without any leading or trailing whitespace and you should be able to find everything you need to know to operate Fluster.
- The render method is set to 'on-change' by default to give users a feel for the application, but you'll almost surely want to change this. This will become <span className="italic">incredibly</span> jumpy when notes contain content that requires a second render pass like math, 3d models, or plots. To get around this there's an option to set this to 'on-save'. This is a little misleading as your notes still save every time you change them, but when this setting is turned on your notes will only render cmd+s is pressed, creating far less distraction and much less needless work for your device.
- Consider using 'topics' and 'subjects' in a sort of heiracheral way to create a layered search. By creating a more broad layer, say 'topics' of things like 'physics', 'math', 'work' and the other of a more narrow category like 'gravity', 'calc2' or 'newProject' you can use one sort of <span className="italic">inside</span> of the other. I think this gives a lot more utility to the various search features in Fluster than just using them directly on top of each other.
- If you don't think you'll need a bibliography, consider adding simple entries for things like websites, with whatever level of specificity you think you'll need. Adding an entry for 'wikipedia' or 'youtube' will obviously be vague enough to apply to a meaningful number of notes in your content and can apply to users with less academic notes. You can even use the website bibliography snippet to cite specific social media channels or blogs, allowing user's to group notes by almost anything.
  - For even more refinement, consider adding a 'title' field to your bibliography entries, even if one isn't already present. This field is what is used for alphabetizing bibliography entries, and all fields without it will be grouped together at "No title found". Of course, you can just filter them...

## Fluster Syntax

### FrontMatter

Insert front matter into your note using yaml syntax seperated by 3 `---`:

```yaml
---
title: My optional note title
---
```


| Property | Description |
| -------- | ----------- |
| title | An optional string that will override the title found through the note's content if present. |
| summary | An optional summary of the note. This will be displayed along with the title in search results. |
| id | An optional id for the note that can be used for dynamic linking via the `[myLink](note:myNoteId)` syntax. |
| ignoreParsers | Used mostly for internal development, can be a list of parsers that will not be applied to the note. Valid options are a list of 'tags', 'citations', 'note_link' and/or 'dictionary'. |

### Tags 

Insert **tags** into your note using the following syntax:

```txt -- title="my_tag_demo.mdx"
This is my tag [[#myTag]] called 'myTag'.
```

### Citations

> Citations are alphabetized by the optional 'title' field. All entries without this field will appear grouped at "No title found".

The citation key must match the bibliography entry key in your bibliography. For example:

```txt
As you can see, according to Ives & Stillwell [[cite:IvesStillwell]]...
```

Will work with the bibliography entry:

```bibtex
@article{IvesStillwell,
    author = {Herbert E. Ives and G. R. Stilwell},
    journal = {J. Opt. Soc. Am.},
    keywords = {Doppler effect; Hydrogen; Light sources; Mirrors; Optical clocks; Transverse effects},
    number = {7},
    pages = {215--226},
    publisher = {Optica Publishing Group},
    title = {An Experimental Study of the Rate of a Moving Atomic Clock},
    volume = {28},
    month = {Jul},
    year = {1938},
    url = {https://opg.optica.org/abstract.cfm?URI=josa-28-7-215},
    doi = {10.1364/JOSA.28.000215},
    abstract = {},
}                
```


### Dictionary Entries

You can insert dictionary entries in any note using the following syntax:

````txt
```dictionary -- My Dictionary Title
This is my dictionary entry.
I can insert any mdx content I like here.
```
````

These entries will automatically collected and inserted into your user defined dictionary, which is why I can't insert a demo here.

### Title
> With an optional subtitle


Since the margin between text can be too much to create some common layouts like a title/subtitle combo, there is a reasonably styled combo component that can be accessed with the following syntax:

```txt
# My title {#myTitleWithAnId}
> My subtitle that follows immediately on the next line
```


<Hint>
Remember that you can create a link that scrolls to a specific section using the syntax `[My markdown link](#myTitleWithAnId){:md}`! More components will accept an id property moving forward, making this link syntax applicable to far more than just headings.
</Hint>

It is important to note though that there must be _no_ whitespace between the title and the block quote. That is what distingishes the behavior between a block quote and a subtitle.


### Checkbox

As in regular markdown, checkboxes can be created with the following syntax:
```md
- [ ] My unchecked item
- [x] My checked item
```

And as in most markdown implemenntations, we can continue our markdown item with additional lines. What's unique about Conundrum is that those line's are compiled into a unique `div`, giving them a seperation within the UI if the developer chooses to display them that way. This can be achieved by:

```md
- [ ] My unchecked item
  My continued item lead by two spaces or 1 tab.
  This line just keeps going until I don't indent 
  or create another list item.
- [x] My checked item
  Of course the same applies here.
```

What's also unique about Conundrum, is that each checkbox can be represented as more than just checked and unchecked, depending on the character used to mark the box as checked.

| Character | Description |
| --------- | ----------- |
| `-` | Pending |
| `?` | Uncertain |
| `x` | Complete | 
| `!` | Important or Error'd |

In a near future release these checkboxes will be completely programmable once the integration with Rhai is complete.

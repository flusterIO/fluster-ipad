## Components




#### Examples

This `Card` component accepts `children`, so we use the `>` closing bracket on the opening tag.

```tsx -- title="components.mdx"
<Card title="My title" desc="My optional description">
This is my card's body.
</Card>
```

This `Toc` (table of contents) component does not accept any children, so we don't need the closing tag and we can write it as:

```tsx -- title="components.mdx"
<Toc />
```



#### Examples

```tsx -- title="strings.mdx"
<Card
    // I can wrap the title with single quotes, 
    // double quotes or backticks.
    title='My title'
    // But because this has an apostrophe, I can't
    // use the single quotes here without escaping it,
    // So I'll use backticks here.
    desc={`This is my description's content.`}
>
...
```


#### _Experimental_ String escaping
> This will be fully supported in a future update. It may or may not work currently...

If we _really_ wanted to use single quotes inside of another pair of single quotes, or double quotes inside of double quotes, we can do that as:

```tsx -- title="strings.mdx"
<Card 
    // If I really need to include matching quotations within the string itself,
    // I have to lead it with a leading `\` character.
    desc={"This is my \"quotation\" here"}
/>
```

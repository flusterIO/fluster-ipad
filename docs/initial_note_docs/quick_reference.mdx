# Quick Reference

## Syntax

### FrontMatter

Insert front matter into your note using yaml syntax seperated by 3 `---`:

```yaml
---
title: My optional note title
---
```


| Property | Description |
| -------- | ----------- |
| title | An optional string that will override the title found through the note's content if present. |
| summary | An optional summary of the note. This will be displayed along with the title in search results. |
| id | An optional id for the note that can be used for dynamic linking via the `[myLink](note:myNoteId)` syntax. |
| ignoreParsers | Used mostly for internal development, can be a list of parsers that will not be applied to the note. Valid options are a list of 'tags', 'citations', 'note_link' and/or 'dictionary'. |


### Tags 

Insert **tags** into your note using the following syntax:

```mdx title="my_tag_demo.mdx"
This is my tag [[#myTag]] called 'myTag'.
```

### Citations

The citation key must match the bibliography entry key in your bibliography. For example:

```mdx
As you can see, according to Ives & Stillwell [[cite:IvesStillwell]]...
```

Will work with the bibliography entry:

```bibtex
@article{IvesStillwell,
    author = {Herbert E. Ives and G. R. Stilwell},
    journal = {J. Opt. Soc. Am.},
    keywords = {Doppler effect; Hydrogen; Light sources; Mirrors; Optical clocks; Transverse effects},
    number = {7},
    pages = {215--226},
    publisher = {Optica Publishing Group},
    title = {An Experimental Study of the Rate of a Moving Atomic Clock},
    volume = {28},
    month = {Jul},
    year = {1938},
    url = {https://opg.optica.org/abstract.cfm?URI=josa-28-7-215},
    doi = {10.1364/JOSA.28.000215},
    abstract = {},
}                
```


### Dictionary Entries

You can insert dictionary entries in any note using the following syntax:

````mdx
```dictionary -- My Dictionary Title
This is my dictionary entry.
I can insert any mdx content I like here.
```
````

These entries will automatically collected and inserted into your user defined dictionary.
